# Task 1.3: PostgreSQL Function for Finding Pricing Tiers

## Overview

This migration creates a PostgreSQL function `find_pricing_tier` that finds the applicable quantity bracket for a given product, variation, and quantity.

## Function Signature

```sql
find_pricing_tier(
  p_product_id UUID,
  p_variation_id UUID,
  p_quantity INTEGER
) RETURNS TABLE (
  tier_id UUID,
  product_id UUID,
  variation_id UUID,
  min_quantity INTEGER,
  max_quantity INTEGER,
  unit_price NUMERIC(10,2)
)
```

## Business Logic

The function implements the following requirements from the design specification:

1. **Quantity Matching**: Returns tiers where `p_quantity >= min_quantity AND (max_quantity IS NULL OR p_quantity <= max_quantity)`

2. **Most Specific Tier**: When multiple tiers match, selects the one with the highest `min_quantity` value (most specific tier)

3. **Variation Handling**: Correctly handles both specific variations and NULL variation_id

4. **Performance**: Uses `STABLE` function type for query optimization and proper indexing

## Usage Examples

### Basic Usage

```sql
-- Find pricing tier for 100 business cards with premium finish
SELECT * FROM find_pricing_tier(
  'product-uuid-here',
  'variation-uuid-here',
  100
);
```

### In Application Code

```typescript
// Using Supabase client
const { data, error } = await supabase.rpc('find_pricing_tier', {
  p_product_id: productId,
  p_variation_id: variationId,
  p_quantity: quantity
})

if (data && data.length > 0) {
  const tier = data[0]
  const totalPrice = tier.unit_price * quantity
  console.log(`Unit price: ₹${tier.unit_price}, Total: ₹${totalPrice}`)
}
```

## Performance Considerations

- Function is marked as `STABLE` to allow query optimization
- Leverages existing indexes on `pricing_tiers`:
  - `idx_pricing_tiers_product_id`
  - `idx_pricing_tiers_variation_id`
- Returns at most 1 row (LIMIT 1) for optimal performance

## Edge Cases Handled

1. **No matching tier**: Returns empty result set
2. **NULL variation_id**: Correctly matches tiers with NULL variation
3. **Overlapping tiers**: Selects most specific (highest min_quantity)
4. **Open-ended tiers**: Handles NULL max_quantity for unlimited upper bound
5. **Boundary values**: Correctly includes quantities at min/max boundaries
